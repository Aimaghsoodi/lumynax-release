Companion products

Two end-user tools that ride on top of the model family — both with live Gradio demos.

🛡 SovereignCode

Data Capsule policy + hash-chained audit ledger

Define a capsule of data residency, allowed purposes, training rules, and human-approval gates. Every request runs through 6 policy gates before any tool call is dispatched. Allowed requests are routed to the best LumynaX model via MaramaRoute.

Live demo ↗ · Package ↗

🧭 MaramaRoute

Sovereign router across all 98 models

Filter on modality, jurisdiction, sovereignty tier, tools, JSON-mode, and context length. Score by quality × sovereignty × cost with task-hint boosting. Returns the best LumynaX model for the request with full reasoning.

Live demo ↗ · Package ↗

Run any model in 3 commands

Every model is fully self-contained — weights, configs, quickstart, requirements, license, Ollama Modelfile, and an HF Space scaffold all ship together.

# Pick any of 98 models from https://huggingface.co/AbteeXAILab
hf download AbteeXAILab/lumynax-chat-hermes-3-llama31-8b-gguf --local-dir hermes-3
cd hermes-3 && pip install -r requirements.txt
python quickstart.py --interactive

Or use the LumynaX CLI:

pip install lumynax
lumynax list                                                 # show all 98 models
lumynax route "fix this Python bug" --local --tools          # pick the best one
lumynax run lumynax-coder-deepseek-v2-lite-16b-gguf -i       # download + run

Sovereignty contract

Every LumynaX repo carries the same release manifest: upstream provenance, license, residency, sovereignty tier, jurisdiction, audit method, and tool/JSON capability — machine-readable for MaramaRoute, human-readable in the model card.

What you get

  • Weights mirrored directly on the AbteeXAILab repo
  • Provenance: upstream HF repo, original license, GGUF mirror used
  • Quickstart that loads local weights (no upstream fetch at runtime)
  • SHA-256 checksums for the runtime scaffold
  • v6 model card: SVG hero, runtime flow, capability bars, mermaid architecture
  • Ollama Modelfile + Gradio Space scaffold

What you can verify

  • Hash-chained audit ledger via SovereignCode
  • Policy gate trace (which gate allowed / denied a request, why)
  • Residency tag (NZ / AU / global) enforced by router
  • Sovereignty tier (1 = remote frontier, 5 = NZ-local) on every model
  • License pinned to upstream — never relicensed by LumynaX
  • Full source on GitHub
